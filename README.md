# Мультиплатформенный Мониторинг & Трекер Арбитража Ставок Финансирования (Funding Rates)

Профессиональное full-stack веб-приложение для обнаружения, мониторинга, исторического анализа и бэктестинга арбитражных стратегий на ставках финансирования (Funding Rates) между 4 ключевыми источниками:
1. **Hyperliquid** (Интервал: 1 час, фокус на криптоактивах)
2. **Variational** (Интервал: 8 часов, фокус на кросс-классах/деривативах)
3. **Extended** (Интервал: 1 час, деривативы с мультивалютным залогом)
4. **Lighter** (Интервал: 1 час, токенизированные рынки)

---

## 🔥 Ключевые Возможности Трекера

1. **Доходные Связки & Спреды (Dashboard)**
   - Автоматический расчет спредов в годовом выражении (**Yield APR %**).
   - Подсчет оптимального направления сделки (на какую биржу заходить в лонг, на какую — в шорт).
   - Фильтрация по типам рынков (все, только крипто, традиционные активы) с учетом кросс-чейн маппинга тикеров.

2. **Интерактивные Графики спредов и накопительного фандинга**
   - Отображение временных рядов ставок финансирования по выбранному активу на интерактивном графике с помощью **Recharts**.
   - Переключение между отображением чистых ставок и кумулятивным эффектом нарастающим итогом.

3. **Расширенный Исторический Трекер с Выбором Дат**
   - **Фильтрация по кастомным диапазонам дат** (`startDate` и `endDate`) или быстрым пресетам (7, 14, 30, 60 дней).
   - Динамический пересчет показателей APR, волатильности ($\sigma$), максимальных и минимальных ставок по каждой бирже в режиме реального времени на основе выбранного временного окна.
   - Клиентский безопасный экспорт всей отфильтрованной исторической базы в формате **CSV** (через HTML5 Blobs).

4. **Симулятор Исторического Бэктестинга (Backtester)**
   - Тестирование торговой стратегии на исторических данных.
   - Настройка кредитного плеча, комиссии за сделку, триггеров входа/выхода по размеру спреда и минимального времени удержания позиции.
   - Расчет подробной статистики: итоговая доходность, Сортино, Шарп, просадка (Drawdown) и лог совершенных сделок.

## 🛠️ Технологический Стек

- **Frontend**: React 18, Vite, TypeScript, Tailwind CSS, Lucide Icons, Recharts для визуализаций, Motion для плавных переходов и интерфейсной анимации.
- **Backend / API**: Node.js, Express, tsx/esbuild для сборки и развертывания сервера.
- **База данных**: Легковесная in-memory СУБД с пре-инсталлированной историей `collected_history.json`.
- **Интеграции**: Мульти-транслятор API ключей для подключения бирж.

---

## 🔑 Управление API-ключами

Ключ для биржи **Extended** обрабатывается на бэкенде:
- Берётся из тела запроса (`req.body.extended_api_key`)
- Считывается из переменных окружения процесса (`process.env.EXTENDED_API_KEY`)

## ⚡ Развертывание локально & Запуск

### Шаг 1: Клонирование и установка зависимостей
```bash
# Установка библиотек
npm install
```

### Шаг 2: Настройка переменных окружения
Создайте файл `.env` в корневой папке и скопируйте значения из `.env.example`:
```env
PORT=3000
EXTENDED_API_KEY=ваш_индивидуальный_ключ
```

### Шаг 3: Запуск в режиме разработки
```bash
npm run dev
```
Сервер запустится на порту `3000` (клиентская часть проксируется через Express Middleware).

### Шаг 4: Сборка в Production
```bash
npm run build
npm start
```

## 📦 Загрузка на GitHub: Инструкция

При публикации этого репозитория на GitHub **ни в коем случае** не загружайте временные бинарные файлы и конфиденциальные данные.

### Какие файлы нужно добавить в `.gitignore`:
Проект уже снабжен преднастроенным файлом `.gitignore`, который автоматически исключает:
- `node_modules/` (папка с зависимостями весит >200MB, устанавливается автономно через `npm i`)
- `.env` (ваши приватные API ключи, пароли и конфигурации)
- `dist/` (готовая продакшн сборка фронтенда и бэкенда)
- Любые отладочные логи (`*.log`)

### Что **НУЖНО** коммитить в git:
```bash
git add package.json package-lock.json      # Зависимости и скрипты сборки
git add tailwind.config.js vite.config.ts   # Конфигурационные файлы сборщиков
git add tsconfig.json                       # Конфигурация TypeScript
git add server.ts                           # Бэкенд на Express
git add src/                                # Исходный код React-клиента
git add index.html                          # Точка входа в DOM
git add .env.example                        # Шаблон переменных окружения
git add README.md                           # Данная документация
```
